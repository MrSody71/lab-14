# Промт 09-3: Проверка и аудит prompt log

## Задача

Убедиться, что для каждого этапа (01–08) существует хотя бы один файл
в `prompts/0N-M-<topic>.md`. Создать отсутствующие файлы с минимальным
содержимым.

## Алгоритм

1. `find prompts/ -name "*.md" | sort` — получить список файлов
2. Для каждого этапа 01–08 проверить наличие хотя бы одного файла
3. Если файл отсутствует — создать с заголовком, датой, коммитом и кратким
   описанием реализованного
4. Для этапов, где файлы уже есть — ничего не трогать

## Минимальный шаблон нового файла

```markdown
# Этап N — <Название>
**Дата:** YYYY-MM-DD
**Ветка:** feat/<topic>
**Коммит:** <hash> (git log --oneline | grep <keyword>)

## Промты использованные в этом этапе
Промты для создания <компонент>.
Полный текст находится в истории чата с Claude Code.

## Что было реализовано
- <пункт 1>
- <пункт 2>
- ...
```
